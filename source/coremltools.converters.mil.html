Unified (TensorFlow and Pytorch)¶

coremltools.converters._converters_entry.convert(model, source='auto', inputs=None, outputs=None, classifier_config=None, minimum_deployment_target=None, convert_to=None, compute_precision=None, **kwargs)¶

Convert TensorFlow or Pytorch models to the Core ML model format. Whether a parameter is required may differ between frameworks (see below). Note that this function is aliased as ct.convert in the tutorials.

Parameters
model:

TensorFlow 1, TensorFlow 2 or Pytorch model in one of the following formats:

For TensorFlow versions 1.x:

  • Frozen tf.Graph

  • Frozen graph (.pb) file path

  • tf.keras.Model

  • HDF5 file path (.h5)

  • SavedModel directory path

For TensorFlow versions 2.x:

  • tf.keras.Model

  • HDF5 file path (.h5)

  • SavedModel directory path

  • A concrete function

For Pytorch:

  • A TorchScript object

  • Path to a .pt file

source: str (optional)

One of [auto, tensorflow, pytorch, milinternal]. auto determines the framework automatically for most cases. Raise ValueError if it fails to determine the source framework.

inputs: list of `TensorType` or `ImageType`
TensorFlow 1 and 2:

  • inputs are optional. If not provided, the inputs are Placeholder nodes in the model (if model is frozen graph) or function inputs (if model is tf function)

  • inputs must corresponds to all or some of the Placeholder nodes in the TF model

  • TensorType and ImageType in inputs must have name specified. shape is optional.

  • If inputs is provided, it must be a flat list.

PyTorch:

  • inputs are required.

  • inputs may be nested list or tuple.

  • TensorType and ImageType in inputs must have name and shape specified.

outputs: list[str] (optional)
TensorFlow 1 and 2:

  • outputs are optional.

  • If specified, outputs is a list of string representing node names.

  • If outputs are not specified, converter infers outputs as all terminal identity nodes.

PyTorch:

  • outputs must not be specified.

classifier_config: ClassifierConfig class (optional)

The configuration if the mlmodel is intended to be a classifier.

minimum_deployment_target: coremltools.target enumeration (optional)

  • one of the members of enum coremltools.target.

  • The value of this parameter determines the type of the model reperesentation produced by the converter. Alternatively, the convert_to argument can be used to specify the model type as well (see description of convert_to argument).

  • if minimum_deployment_target <= coremltools.target.iOS14/

    coremltools.target.macOS11/ coremltools.target.watchOS7/ coremltools.target.tvOS14:

    • a model of type neural network is produced.

  • if minimum_deployment_target >= coremltools.target.iOS15/

    coremltools.target.macOS12/ coremltools.target.watchOS8/ coremltools.target.tvOS15:

    • a model of type ML Program is produced.

  • if this argument is specified and convert_to is also specified then they must be compatible.

    Invalid values are (convert_to=”neuralnetwork”, minimum_deployment_target=coremltools.target.iOS15) or (convert_to=”mlprogram”, minimum_deployment_target=coremltools.target.iOS14) etc.

  • When both minimum_deployment_target and the convert_to arguments are unspecified, converter aims to produce the neural network model type with as minimum of a deployment target as possible.

convert_to: str (optional)

  • Must be one of ['neuralnetwork', 'mlprogram', 'milinternal'].

  • The value of this parameter determines the type of the model reperesentation produced by the converter. Alternatively, the minimum_deployment_target argument can be used to specify the model type as well (see description of the minimum_deployment_target argument).

  • 'neuralnetwork': Returns MLModel (i.e. of type coremltools.models.MLModel) containing a NeuralNetwork proto. This is the older Core ML format and the model saved from this returned object is either, depending on the layers used in the model, executable on iOS13/macOS10.15/watchOS6/tvOS13 and above, or ececutable on iOS14/macOS11/watchOS7/tvOS14 and above.

  • 'mlprogram' : Returns MLModel (i.e. of type coremltools.models.MLModel) containing a MILSpec.Program proto. This is the newer Core ML format, and the Core ML model saved from this returned object is executable on iOS15, macOS12, watchOS8, tvOS15.

  • 'milinternal': Returns MIL program object (i.e. of type coremltools.converters.mil.Program).

    MIL program is primarily used for debugging and inspection. It can be converted to MLModel for execution, by using ct.convert(mil_program, convert_to=”neuralnetwork”) or ct.convert(mil_program, convert_to=”mlprogram”)

  • When both minimum_deployment_target and the convert_to arguments are unspecified, converter aims to produce the neural network model type with as minimum of a deployment target as possible. (see description of the minimum_deployment_target argument).

compute_precision: coremltools.precision enumeration or of type ct.transform.FP16ComputePrecision() (optional)
  • Must be one of the following:
    • coremltools.precision.FLOAT16

      • in this case the transform “coremltools.transform.FP16ComputePrecision(op_selector= lambda op:True)” will be applied, which injects “cast” ops to convert the float32 dtypes of intermediate tensors to float16

    • coremltools.precision.FLOAT32

      • in this case, no transform is applied and hence the original float32 tensor dtype in the source model is preserved

    • coremltools.transform.FP16ComputePrecision(op_selector=…)

      • this can be used to control which tensors get cast to float16.

      • e.g.: coremltools.transform.FP16ComputePrecision(op_selector= lambda op: op.op_type != “linear”) will cast all the float32 tensors to be float16, except the input/output tensors to any “linear” op.

  • If None, defaults to coremltools.precision.FLOAT32

  • TODO: rdar://74140243.

    • Before coremltools 5.0 release, change the default to coremltools.precision.FLOAT16 when convert_to=”mlprogram”

Returns
model: coremltools.models.MLModel or coremltools.converters.mil.Program

A Core ML MLModel object or MIL Program object (see convert_to)

Examples

TensorFlow 1, 2 (model is a frozen graph):

>>> with tf.Graph().as_default() as graph:
>>>     x = tf.placeholder(tf.float32, shape=(1, 2, 3), name="input")
>>>     y = tf.nn.relu(x, name="output")

# Automatically infer inputs and outputs

>>> mlmodel = ct.convert(graph)
>>> test_input = np.random.rand(1, 2, 3) - 0.5
>>> results = mlmodel.predict({"input": test_input})
>>> print(results['output'])

TensorFlow 2 (model is tf.Keras model path):

>>> x = tf.keras.Input(shape=(32,), name='input')
>>> y = tf.keras.layers.Dense(16, activation='softmax')(x)
>>> keras_model = tf.keras.Model(x, y)

>>> keras_model.save(h5_path)
>>> mlmodel = ct.convert(h5_path)

>>> test_input = np.random.rand(2, 32)
>>> results = mlmodel.predict({'input': test_input})
>>> print(results['Identity'])

Pytorch:

>>> model = torchvision.models.mobilenet_v2()
>>> model.eval()
>>> example_input = torch.rand(1, 3, 256, 256)
>>> traced_model = torch.jit.trace(model, example_input)

>>> input = ct.TensorType(name='input_name', shape=(1, 3, 256, 256))
>>> mlmodel = ct.convert(traced_model, inputs=[input])
>>> results = mlmodel.predict({"input": example_input.numpy()})
>>> print(results['1651']) # 1651 is the node name given by PyTorch's JIT

See neural-network-conversion for more advanced options.
